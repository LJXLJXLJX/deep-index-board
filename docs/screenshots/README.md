# Screenshots

Use this folder for README and release screenshots.

Recommended set:

1. `main-workflow.png`
   - Show the history list, collapsed or expanded favorites, and the editable text preview.
   - This should be the primary README screenshot.

2. `image-workflow.png`
   - Show image preview, OCR result, or semantic image search context.
   - Use this only if it clearly shows something different from a normal clipboard manager.

3. `macos-permissions.png`
   - Optional. Only add this if the unsigned `.app` / Accessibility permission flow becomes a common support issue.

Avoid making one screenshot per feature. A GUI tool usually benefits more from one clear workflow shot than a gallery of small, repetitive states.
